# 0.1 이 책을 읽는 법

::: lead
이 책의 목표는 하나다. **파이썬으로 코딩테스트를 통과하면서, 동시에 ROS·ML·DL·비전·AI를 직접 만들 수 있는 개발자**가 되는 것. 이 절은 그 목표까지 가는 지도이자 사용 설명서다. 5분이면 읽는다. 건너뛰지 마라 — 이 책을 어떻게 쓰느냐가 도달 지점을 결정한다.
:::

## 이 책이 서 있는 자리

시중의 파이썬 책은 대개 세 종류다.

| 종류 | 다루는 것 | 빠진 것 |
| --- | --- | --- |
| 입문서 | 문법, 자료형, 간단한 예제 | 왜 그렇게 동작하는지, 실무 규모 |
| 알고리즘 책 | 자료구조, 풀이 패턴 | 파이썬 자체의 깊이, 응용 분야 |
| 분야별 전문서 (ML/비전/ROS) | 해당 라이브러리 | 언어의 기반, 다른 분야와의 연결 |

셋을 다 사서 읽으면 될 것 같지만, 실제로는 잘 안 된다. **연결이 끊기기 때문**이다. 입문서에서 배운 리스트가 왜 NumPy 배열보다 느린지, 알고리즘 책의 재귀가 왜 ROS 콜백에서 문제를 일으키는지, 딥러닝 책의 `DataLoader`가 왜 `num_workers`를 늘려도 안 빨라지는지 — 이런 질문의 답은 세 책 **사이**에 있다.

이 책은 그 사이를 메우려고 쓴다. 그래서 순서가 이렇다.

::: note 이 책의 논리 구조
**언어의 코어**(Part I~III)를 이해하면 → **동시성·성능**(Part IV~V)이 왜 그런지 보이고 → 그 위에서 **자료구조·알고리즘**(Part VII~VIII)이 파이썬 관용구로 자연스러워지며 → **NumPy**(Part IX)가 왜 필요한지 몸으로 알게 되고 → 그 위에 **ML·DL·비전**(Part X~XII)이 얹히고 → 마지막으로 **ROS·AI**(Part XIII~XIV)가 이 모두를 조립한다.
:::

## 지금 당신의 위치

이 책은 **파이썬 기본 문법을 이미 아는 사람**을 기준으로 쓴다. `for` 문과 `if` 문, 리스트와 딕셔너리를 만들 줄 알고, 함수를 정의해 봤다면 충분하다.

그렇다고 Part I을 건너뛰지 마라. Part I은 "리스트 만드는 법"을 가르치지 않는다. **리스트가 메모리에서 어떻게 생겼길래 `append`는 빠르고 `insert(0, x)`는 느린지**를 다룬다. 아래 코드가 왜 이런 결과를 내는지 3초 안에 설명할 수 있다면 1.1절을 훑고 넘어가도 좋다. 못 하겠다면, 거기가 당신의 출발점이다.

```pyrepl
>>> a = [1, 2, 3]
>>> b = a
>>> b += [4]
>>> a
[1, 2, 3, 4]
>>> c = [1, 2, 3]
>>> d = c
>>> d = d + [4]
>>> c
[1, 2, 3]
```

::: answer 답이 궁금하다면
`+=`는 리스트에서 `__iadd__`를 호출해 **원본을 제자리에서 수정**한다. `b`와 `a`는 같은 객체를 가리키므로 `a`도 바뀐다. 반면 `d = d + [4]`는 `__add__`로 **새 리스트를 만들어** `d`라는 이름에 다시 묶는다. `c`가 가리키는 객체는 건드리지 않았다.

핵심은 *"변수는 값을 담는 상자가 아니라 객체에 붙이는 이름표"* 라는 것. [1.1 객체, 이름, 참조](#/objects-names)에서 이 모델을 끝까지 파고든다.
:::

## 이 책의 표기 규칙

읽는 데 필요한 약속들이다.

### 코드 블록

실행 가능한 완결된 코드는 이렇게 나온다. 오른쪽 위 **복사** 버튼으로 바로 가져갈 수 있다.

```python title="greet.py"
def greet(name: str) -> str:
    """인사말을 만든다."""
    return f"안녕하세요, {name}님"


if __name__ == "__main__":
    print(greet("파이썬"))
```

대화형 세션(REPL)은 프롬프트 `>>>` 와 함께 나온다. 여기서 **`>>>` 없는 줄은 출력**이다. 직접 쳐 볼 때 프롬프트는 빼고 입력하면 된다.

```pyrepl
>>> greet("세상")
'안녕하세요, 세상님'
```

터미널 명령은 이렇게 표시한다.

```bash
uv run python greet.py
```

주목해야 할 줄은 강조해 둔다. 아래에서는 3번 줄이 핵심이다.

```python title="함정: 가변 기본값" {3}
def add_item(item, bucket=[]):
    # 기본값 리스트는 함수 정의 시점에 딱 한 번 만들어진다.
    bucket.append(item)          # 호출할 때마다 같은 리스트가 커진다
    return bucket
```

본문 흐름을 끊지 않으면서 따로 봐야 할 것들은 상자에 담는다.

알아두면 좋은 보충 설명. 몰라도 진도는 나가지만 알면 이해가 깊어진다.

:::

CPython 내부 구현이나 PEP 같은 한 단계 아래 이야기. 처음 읽을 때는 건너뛰어도 된다. 두 번째 읽을 때 여기가 재미있어진다면 실력이 붙은 것이다.
:::

:::

::: quiz 연습문제

:::

::: hist 배경
왜 이렇게 설계됐는지에 대한 역사. 맥락을 알면 잘 안 잊힌다.

### 용어

기술 용어는 **한국어 + 영어 병기**로 처음 나올 때 한 번 표기한다. 예: 참조 계수(reference counting), 구조적 패턴 매칭(structural pattern matching). 이후에는 문맥에 따라 편한 쪽을 쓴다. 영어 문서와 논문을 읽어야 하는 분야이므로 원어를 지우지 않는다.

## 어떻게 읽을 것인가

### 원칙 1 — 손으로 치지 않은 코드는 읽지 않은 것이다

이 책의 코드는 전부 실행 가능하다. **복사 버튼을 누르지 말고 직접 쳐라.** 느리고 답답하지만, 이게 제일 빠르다. 오타를 내고, 에러 메시지를 읽고, 고치는 과정에서 배우는 것이 읽기만 할 때의 몇 배다.

특히 **결과를 예측한 뒤에 실행하라.** "이건 `[1, 2, 3, 4]`가 나올 거야" 하고 엔터를 친다. 맞으면 지식이 확인되고, 틀리면 그 순간이 가장 크게 배우는 순간이다. 그냥 실행해서 결과를 보면 아무것도 남지 않는다.

### 원칙 2 — 막히면 표시하고 넘어간다

Part I의 메타클래스나 Part IV의 free-threaded 같은 주제는 처음에 안 와닿는 게 정상이다. 각 절 맨 아래 **‘이 절을 완료로 표시’** 버튼이 있고, 왼쪽 목차에 체크(✓)가 남는다. 완벽히 이해한 절만 체크하고, 나머지는 비워 둔 채 진도를 나가라. 뒤쪽을 읽다 보면 앞쪽이 저절로 풀리는 일이 매우 흔하다.

::: tip 검색을 적극적으로 써라
상단 돋보기 또는 <kbd>/</kbd> 키로 **전체 본문 검색**이 열린다. "다익스트라", "GIL", "브로드캐스팅" 처럼 개념 이름으로 찾으면 그 개념을 다루는 모든 절이 나온다. 이 책은 앞에서 뒤로 읽는 것만큼이나 **필요할 때 찾아 읽는 레퍼런스**로 쓰라고 만들었다.
:::

### 원칙 3 — 목적에 맞는 경로를 탄다

전체를 순서대로 읽는 게 이상적이지만, 급한 목표가 있다면 아래 경로를 쓴다.

**코딩테스트가 3개월 안에 있다면**

1. Part VII 전체 (자료구조·알고리즘) — 여기가 본체다
2. Part VIII 전체 (코딩테스트 실전) — 특히 8.2 입출력, 8.3 시간 초과
3. 병행: Part I 중 1.3 시퀀스, 1.6 dict, 1.9 컴프리헨션, 1.18 이터레이터
4. 전 파트의 🎯 **코딩테스트 포인트** 상자만 훑기

**로봇/ROS가 급하다면**

1. Part I 전체 → 1.10~1.18은 특히 중요 (콜백과 제너레이터를 이해해야 한다)
2. Part IV (동시성) — ROS 노드는 결국 동시성 문제다
3. Part IX 9.1~9.3 (NumPy)
4. Part XII 12.1~12.7 (고전 비전, 캘리브레이션까지)
5. Part XIII 전체

**ML/DL이 급하다면**

1. Part IX 전체 (NumPy·pandas)
2. Part X (머신러닝) → Part XI (딥러닝)
3. 되돌아와서 Part V (성능), Part IV (동시성) — 학습 병목을 잡으려면 반드시 필요하다

::: warn 지름길의 대가
경로를 타면 빨리 도달하지만, 반드시 **되돌아와서 메운다**. Part I~V를 건너뛰고 Part XI로 간 사람은 `loss`가 `nan`이 되는 순간, `CUDA out of memory`가 뜨는 순간 멈춘다. 라이브러리 사용법은 검색으로 해결되지만 **왜 그런지**는 검색으로 안 나온다. 그 차이가 이 책의 존재 이유다.
:::

## 실습 환경

Part 0의 나머지 절에서 환경을 제대로 세운다. 지금은 결론만 알아 두자.

- **파이썬 3.14** 기준으로 쓴다. 3.10 이상이면 대부분 그대로 동작하고, 버전이 갈리는 지점은 그때마다 표시한다.
- 패키지 관리는 **uv**를 쓴다. `pip`, `venv`, `poetry`, `conda`를 모두 대체한다. ([0.3 uv](#/uv))
- 린터·포매터는 **ruff**, 타입체커는 **pyright**. ([0.4 도구 세팅](#/tooling))
- Part XI 이후(딥러닝·비전)는 **GPU가 있으면 좋지만 없어도 된다.** 없는 경우의 대안(Colab, CPU 축소판)을 그때 함께 안내한다.

::: note 버전 표기 규칙
`3.12+` 처럼 쓰면 "3.12 이상에서만 동작"이라는 뜻이다. `3.11에서 추가` 는 그 버전부터 생겼다는 뜻이고, 그 이전 버전용 대안이 있으면 함께 적는다.
:::

## 이 앱 자체에 대해

이 책은 **HTML 파일 하나와 마크다운 묶음**으로 되어 있다. 서버도, 인터넷도, 빌드 도구도 필요 없다.

- **왼쪽 목차**: 부(Part)를 접었다 펼 수 있고, 위쪽 칸에서 목차를 걸러낼 수 있다. 완료한 절에는 ✓ 가 붙는다.
- **오른쪽 목차**: 지금 보는 절 안에서의 위치. 화면이 좁으면 숨는다.
- **진도**: 왼쪽 위 원형 그래프. 브라우저에 저장되므로 PC와 휴대폰은 각각 따로 집계된다.
- **테마**: 상단 오른쪽 아이콘으로 시스템/밝게/어둡게 순환. 글자 크기도 그 옆에서 조절한다.
- **단축키**: <kbd>/</kbd> 또는 <kbd>Ctrl</kbd>+<kbd>K</kbd> 검색, <kbd>[</kbd> 이전 절, <kbd>]</kbd> 다음 절, <kbd>Esc</kbd> 닫기.

::: tip 휴대폰에서 보기
PC에서 `python build.py --serve` 를 실행하면 접속 주소 두 개가 뜬다. 같은 와이파이에 연결된 휴대폰 브라우저에서 `http://192.168.x.x:8800` 형태의 주소를 열면 된다. 홈 화면에 추가해 두면 앱처럼 쓸 수 있다.
:::

## 시작하기 전에

마지막으로 하나만.

이 책은 길다. 160개가 넘는 절이 있고, 전부 읽으면 웬만한 전공 서적 몇 권 분량이다. **끝까지 읽는 것이 목표가 아니다.** 목표는 *"모르는 것이 나왔을 때 어디를 봐야 할지 아는 상태"* 그리고 *"필요하면 밑바닥까지 내려갈 수 있다는 자신감"* 이다.

전문가와 초보자의 차이는 아는 양이 아니라 **모르는 것을 다루는 방식**에 있다. 초보자는 모르면 검색해서 붙여넣고 넘어간다. 전문가는 모르면 한 층 아래로 내려가서 확인하고 올라온다. 이 책은 그 "한 층 아래"를 매번 같이 내려가 주는 책이다.

그럼 시작하자.
